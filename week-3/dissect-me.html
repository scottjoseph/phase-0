October 25, 2015

What are the benefits of version control?

So first off, what is version control. Version Control is a system that tracks updates or changes to files so that the user can go back to any specific version at a later time. If you play video games, it's like a check point or save point. When you die in the game or you make a mistake with something, you either go back to the check point or load up a save file and do something else to continue on with your game that way you want it. Version control is just that, a system that saves at points in time that allow you to load up whenever you feel the need to such as if you make a mistake or want to change something about a file or series of files. The benefit of it is that it saves time so that you don't have to basically start over.

How does git help you keep track of changes?

Git is a great version control tool. The reason it's a great tool is because it keeps track of the changes you make for you. It's able to tell you what what file(s) have been changed, which ones haven't, and in the event that you like where you're at in your work, allows you to make a "commit" which is git's version of a save point. Easy way to think about it is "You're committing to the work you've done up to this point".

Why use GitHub to store your code?

Github is a great place to store code because it allows others to see it, copy it, work on it, and give back suggestions they make. It allows for code to grow into something much bigger, much faster. It's like working on a school project in groups, except your group is more than a few classmates, it's every student in the world. It allows you to share what you have and others can share what they have or even learn from you.
